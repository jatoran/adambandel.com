Problem

Bookmarks and watchlists are everywhere and nowhere: links pile up across browsers, notes apps, text messages, and YouTube “Watch Later,” but they rarely turn into action. Titles are inconsistent, sites rot, and there’s no unified way to search, filter, or rediscover what you once intended to read or watch. Existing read‑it‑later tools focus on consumption and syncing; I wanted a single‑user, self‑hosted system for intent capture, LLM‑powered enrichment, and discovery across everything I personally care about.

Approach

Personal Media Feed (PMF) is a localhost‑first web app that treats each link (or manual note) as a MediaItem with rich, structured metadata. When you add an item, the backend enriches it using a configurable LLM pipeline and stores an embedding in ChromaDB for semantic search and related‑item discovery. A modern React UI provides fast filtering, bulk actions, previews, and a RAG‑backed “Ask Your Feed” chat that pulls context from your own corpus.

Architecture

  • Single user, self‑hosted on localhost; no auth or multi‑tenancy.
  • Backend: FastAPI + SQLModel on SQLite (async), SSE for live updates.
  • Enrichment: scrape + LLM for generic URLs; YouTube path uses API + LLM cleaning.
  • Embeddings: ChromaDB with metadata filters; admin rebuild and re‑enrich actions.
  • RAG chat: sessioned history in SQLite; retrieval from Chroma + item details from SQL.
  • Smart Views: saved filter/sort presets surfaced in the UI.
  • Monitoring: API usage/tokens/costs logged and prunable.
  • Settings: pick LLMs per task and edit prompts; stored in DB, keys in .env.

Stack

  • FastAPI, SQLModel, aiosqlite — Async API + typed models on SQLite.
  • ChromaDB — Persistent vector store for semantic search and related items.
  • OpenAI/OpenRouter — Configurable LLMs for enrichment, chat, embeddings.
  • BeautifulSoup, trafilatura — Extraction and readability for article content.
  • React 19, React Query, Zustand — Data fetching and state management.
  • Tailwind v3 + Shadcn UI — Productive, consistent component styling.

Challenges

  • Reliable enrichment — Mixed quality across sites and models; mitigated with editable fields, configurable prompts/models, and a clean YouTube flow.
  • Balancing cost vs. quality — Tracks token usage and estimated cost; lets me choose lighter or heavier models per task.
  • Schema evolution — During development, I used a wipe‑and‑recreate strategy for SQLite + Chroma, with Alembic scaffolded for later.

Outcomes

PMF centralizes my intent backlog and makes it searchable beyond keywords. I can add URLs in bulk, get instant enrichment via SSE updates, filter by mood, length, tags, or status, and open a minimal reading mode for articles. The RAG chat helps answer questions like “What were those longform interviews on ML safety I saved last month?” while the Discover view surfaces semantically related items I might have forgotten.
