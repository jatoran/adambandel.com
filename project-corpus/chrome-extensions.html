Problem

Long LLM chats get unwieldy fast. It’s easy to lose your place in a thread, hard to visualize branching ideas, and slow to iterate when you must wait for each response before typing the next prompt. I built a small suite of Chrome extensions to improve flow while working in ChatGPT and Claude.

Approach

Each extension focuses on one sharp capability and runs as a Manifest V3 content script with a minimal UI and near‑zero overhead. Two “Queue” extensions automate prompt sequencing for ChatGPT and Claude; a “Minimap” gives fast spatial navigation through long chats; and a “Tree View” visualizes conversation structure with an interactive D3 graph. Common patterns include resilient DOM targeting with multiple selectors, guarded async send flows, and local persistence via chrome.storage.local for state and preferences.

Extensions

  • ChatGPT Queue — Queue multiple prompts and auto‑send them sequentially as ChatGPT finishes generating. Intercepts Enter (vs Shift+Enter for newlines), shows a lightweight in‑page queue indicator, and guards against overlapping sends by detecting the “Stop generating” state. Uses a background service worker and chrome.runtime messaging to manage status.
  • Claude Queue — Same concept, adapted for Claude.ai’s ProseMirror editor and UI. Employs multiple fallback selectors for the input area and send button, and only advances the queue when the model is idle.
  • Scroll Minimap (ChatGPT/Claude) — A scaled clone of the chat container rendered in a fixed overlay for quick navigation. Includes a toggle and refresh controls, color‑cues user vs assistant turns, and maps minimap drag/scroll to the main thread. Designed to be light and non‑intrusive.
  • Conversation Tree (ChatGPT) — A D3‑powered, draggable, resizable graph of the current conversation. Highlights the visible branch, offers compact “short link” mode, tooltips with clipped message text, a glass/transparent background option, and an optional periodic auto‑refresh. Trees persist per conversation ID in chrome.storage.local.

Architecture

  • MV3 content scripts + optional background service workers.
  • Robust DOM targeting with layered selectors for evolving UIs (textarea, send/stop buttons, conversation containers).
  • Safe sequencing: sendingInProgress flags and realtime detection of loading/stop states prevent duplicate sends.
  • Local state: queues and user preferences persisted in chrome.storage.local and synced across tabs via chrome.runtime messaging.
  • Visualization: D3 v7 builds an interactive tree; layout and tooltips tune performance with character limits and incremental refresh.

Stack

  • Chrome Extensions (MV3) — Content scripts, service workers.
  • JavaScript + CSS — Minimal, dependency‑light UIs.
  • D3.js v7 — Conversation tree rendering and interaction.
  • Chrome APIs — storage, runtime, tabs, scripting.

Challenges

  • Fragile selectors — Chat UIs change frequently. Solution: maintain multiple fallbacks per target and detect platform (ChatGPT vs Claude) at runtime.
  • Send concurrency — Prevent sending while the model is generating. Solution: detect “Stop generating”/loading states and gate sends with explicit flags and retries.
  • Performance/UX — Keep overlays responsive during long chats. Solution: scale‑down clones for the minimap, character limits and optional refresh intervals for the tree, and passive UI controls.

Outcomes

In day‑to‑day use these tools reduce friction: queueing lets you stack prompts and step away; the minimap turns long threads into a glanceable index; and the conversation tree surfaces structure you can act on. Collectively they make research and iteration faster with fewer context slips.
