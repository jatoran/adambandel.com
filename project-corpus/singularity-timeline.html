Problem

AI progress spans hardware, algorithms, data ecosystems, neuroscience, and robotics — and it’s accelerating. Static charts flatten nuance: we lose category context, density over time, and the sensation of “inward pressure” as milestones converge on an ASI horizon. The goal is a fast, explorable visualization that scales, conveys chronology and category structure, and invites discovery.

Approach

Singularity Timeline renders milestones as an interactive spiral galaxy. Categories form colored arms; each year maps to a radius (outer = earlier, inner = later). Events sit inside yearly bands, with gentle declutter to prevent overlap. A center node represents Artificial Superintelligence (ASI). UI overlays provide instant detail, category focus, zoom, and a time‑warp control to bias perception (linear / log / √t) without moving the underlying data.

Architecture

  • Rendering — React‑Three‑Fiber on top of three.js; InstancedMesh for nodes and density dots to keep draw calls low.
  • Spiral geometry — radius by year via calculateRadiusForYear(); category arms via armAngle() with 2 turns.
  • UI overlays — Tooltip, InfoPanel, Legend, Zoom, Time‑Warp rendered as React DOM on top of WebGL.
  • State — Zustand slices (dataSlice, uiSlice, vizSlice) with selector‑driven reactivity.
  • Picking & interaction — InstancedMesh pointer events map instanceId → eventId; hover and select update the stores.
  • FLOPs density — ring‑based dots with a small GLSL program; alpha scales with per‑year log‑density.
  • ETL pipeline — nested source JSON transformed to a flat dataset for fast client queries.
  • Extensibility — alternative scenes (Helix, Cylinder, Funnel) stubbed; list route planned for virtualized browsing.

Stack

  • React 18 + TypeScript — app shell, overlays, routing.
  • three.js + React‑Three‑Fiber + drei — camera, controls, lines, HTML in 3D, and GPU instancing.
  • Zustand — tiny global state with ergonomic selectors; camera commands and UI toggles route through the store.
  • D3 (force/quadtree) — earmarked for worker‑based declutter and hit‑testing windows.
  • Vite 6 + Vitest + ESLint — modern tooling, fast dev server, typed tests for math and stores.

Challenges

  • Visual clarity — sibling events in the same year/category can collide; a light collision nudge keeps them legible.
  • Perceptual mapping — reconciling a satisfying spiral with year spacing; time‑warp biases angle without breaking chronology.
  • Overlay depth — mixing DOM labels and WebGL can hide labels; careful z‑layering and drei <Html> keep them readable.
  • Performance headroom — instancing and memoization keep the render budget low; workers are planned for local declutter.

Outcomes

The spiral scene, category arms, yearly bands, ASI center, tooltip, info panel, legend, zoom controls, and time‑warp are implemented. FLOPs density rings render with an additive shader for a subtle “activity halo.” Alternative scenes and a virtualized list view are scaffolded for future iterations. The ETL flow produces a TimelineDataset consumed on boot, making the app data‑driven and easy to extend.
