Overview

TidyOps is a single binary that dispatches to a plugin registry per workspace. It aims to codify “runbook” chores like linting, formatting, license checks, and dependency drift detection across many repositories with a consistent interface.

Key Features

  • Plugin discovery via tidyops.json in each repo.
  • Parallel execution with per-task timeouts and retries.
  • Structured JSON logs + pretty TTY reporter.
  • Fail-fast with allowed failure list.

Example Config

{
  "plugins": [
    { "use": "lint/eslint", "cwd": "apps/web" },
    { "use": "format/prettier", "cwd": "." },
    { "use": "licenses/check", "allow": ["MIT", "Apache-2.0"] }
  ]
}

CLI Snippets

$ tidyops run --only lint --since HEAD~1
▶ 3 tasks matched (eslint, typing, stylelint)
✔ eslint (apps/web) — 12.3s
✔ typing (packages/ui) — 5.1s
✖ stylelint (apps/docs) — 7.2s (allowed failure)
summary: 2/3 successful (allowed: 1)
